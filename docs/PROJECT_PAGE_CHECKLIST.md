# Checklist — Page projet et préparation Play Store

Cette checklist accompagne la page publique `public/projet.html`. Les cases interactives visibles sur la page sont enregistrées uniquement dans le navigateur ; ce document reste la source de suivi partagée dans le dépôt.

## Expérience de la page projet

- [x] Cash Pulse interactif conservé et responsive.
- [x] Atlas interactif relié à l’inventaire généré.
- [x] Trois parcours narratifs : pilotage mensuel, voyage partagé et santé.
- [x] Mini-démos Transaction, Trip, Sport et Nutrition.
- [x] Résumé éditorial des cinq dernières versions.
- [x] Navigation directe vers la politique de confidentialité.
- [ ] Faire relire la page par deux utilisateurs externes.
- [ ] Vérifier les textes anglais avec un relecteur natif.

## Confidentialité et conformité

- [x] Politique de confidentialité publique FR/EN créée.
- [x] Demande externe de suppression de compte accessible.
- [x] Lien vers la confidentialité présent dans l’application.
- [ ] Tester une suppression complète sur un compte de démonstration.
- [ ] Publier l’URL HTTPS définitive de la politique.
- [ ] Reporter exactement les traitements dans le formulaire Data safety Google Play.
- [ ] Faire valider le contenu juridique par une personne qualifiée.

## Publication Android

- [x] APK de test versionnée disponible.
- [x] Permissions Android limitées à Internet et notifications.
- [ ] Produire et signer un Android App Bundle de production.
- [ ] Préparer icône, bannière et captures Play Store finales.
- [ ] Ajouter l’URL Play Store dès que la fiche est publiée.
- [ ] Terminer une campagne de test fermé et centraliser les retours.

## Qualité et exploitation

- [x] Build Vite et contrôles de syntaxe disponibles.
- [x] Inventaire Atlas et documentation vérifiables par script.
- [ ] Tester la page projet sur téléphone, tablette et desktop réels.
- [ ] Ajouter une vérification automatique des liens publics.
- [ ] Mettre à jour la checklist à chaque publication majeure.
